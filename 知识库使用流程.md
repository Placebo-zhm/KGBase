# 知识库使用流程

## 使用目标

这份文档说明日常如何使用知识库。目录结构和分类规则见 [[知识库结构设计]]，模板规范见 [[98-Templates/模板说明|模板说明]]，Git 备份和同步流程见 [[Git管理指南]]。

## 快速记录

1. 新想法先进入 `00-Inbox`。
2. 使用 [[98-Templates/临时笔记模板|临时笔记模板]] 快速记录。
3. 不在记录时纠结分类。
4. 保留上下文，方便后续整理。

## 每日整理

1. 清理当天新增笔记。
2. 给重要笔记补充标题、链接和标签。
3. 将临时笔记移动到合适目录。
4. 对完成的整理执行 Git 提交。

## 每周整理

1. 清空或压缩 `00-Inbox`。
2. 更新重要 MOC。
3. 检查正在进行的项目。
4. 将完成或暂停的项目移动到 `40-Archives`。
5. 检查根目录是否有散落附件。
6. 将常用格式沉淀到 `98-Templates`。

## 链接和标签

优先使用双链连接具体笔记：

```markdown
[[Git - 常用命令]]
[[Obsidian - 双链使用方法]]
```

标签用于状态或横向属性，不建议代替目录和 MOC。

推荐标签：

```text
#todo
#review
#idea
#source/book
#source/paper
#source/video
```

避免创建过多近义标签，例如同时使用 `#AI`、`#人工智能`、`#机器智能` 表示同一件事。

## 最小可执行规则

如果不想维护太复杂，只需要坚持以下规则：

1. 新内容先进入 `00-Inbox`。
2. 每周清理一次 Inbox。
3. 项目放 `10-Projects`，资料放 `30-Resources`，长期方向放 `20-Areas`。
4. 每个重要主题至少有一个 MOC。
5. 常用格式沉淀为 `98-Templates` 中的模板。
6. 附件统一放 `99-Attachments`。
7. 每次完成整理后提交到 Git。
