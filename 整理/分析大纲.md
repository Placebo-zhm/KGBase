# 一、先判断：它到底是什么？
- 它本质上是什么？
    
- 它是一个新技术，还是旧技术的新组合？
    
- 它是工具、平台、系统、协议、硬件，还是服务？
    
- 它解决的是技术问题，还是商业问题？
    
- 它和已有方案相比，到底新在哪里？
    
例如分析 ChatGPT，不能只说“AI 聊天机器人”，还要看它背后的大语言模型、人机交互方式、应用生态和商业入口。
# 二、它解决了什么问题？

技术一定要回到问题本身。

可以问：
- 用户原来遇到什么痛点？
    
- 这个痛点是真痛点，还是伪需求？
    
- 它让什么事情变得更快、更便宜、更简单、更自动化？
    
- 它替代了谁的工作流程？
    
- 没有它，用户会用什么替代方案？
    
一个技术产品如果说不清楚“替代了什么”，通常价值就不够清晰。
# 三、它的核心原理是什么？

这是分析技术产品最重要的一层。

可以问：

- 它背后的核心技术是什么？
    
- 技术链路是怎样的？
    
- 输入是什么，处理过程是什么，输出是什么？
    
- 它依赖哪些算法、芯片、数据、模型、传感器、网络或系统？
    
- 哪个技术环节最关键？
    
- 它的性能瓶颈在哪里？
    
可以用一个简单结构：

**输入 → 处理 → 输出 → 反馈**

比如自动驾驶：

**摄像头/雷达输入 → 感知模型识别环境 → 决策系统规划路线 → 控制系统操作车辆 → 实时反馈修正。**
# 四、它的产品形态是什么？

技术不等于产品。很多技术很强，但产品不好用。

可以问：

- 用户实际接触到的界面是什么？
    
- 它是 App、网页、设备、插件、API，还是系统能力？
    
- 用户使用路径是否简单？
    
- 学习成本高不高？
    
- 体验是否稳定？
    
- 它是面向个人用户、企业客户，还是开发者？
    
- 它是单点工具，还是完整工作流？
    
例如很多 AI 工具的技术很强，但产品失败在：入口复杂、结果不可控、用户不知道什么时候该用。
# 五、它的关键指标是什么？

不同技术产品有不同评价标准。

如果是软件产品，可以看：

- 活跃用户数
    
- 留存率
    
- 使用频次
    
- 转化率
    
- 响应速度
    
- 稳定性
    
- 成本
    
- 用户满意度
    

如果是 AI 产品，可以看：

- 准确率
    
- 幻觉率
    
- 推理能力
    
- 延迟
    
- token 成本
    
- 上下文长度
    
- 可控性
    
- 安全性
    

如果是硬件产品，可以看：

- 性能
    
- 功耗
    
- 续航
    
- 重量
    
- 良率
    
- 成本
    
- 供应链稳定性
    
- 维修难度
    
如果是平台型产品，可以看：

- 生态规模
    
- 开发者数量
    
- 网络效应
    
- API 使用量
    
- 第三方应用数量
    
- 迁移成本
    
# 六、它的商业模式是什么？

技术产品不能只看技术，还要看能不能赚钱。

可以问：

- 谁付钱？
    
- 为什么愿意付钱？
    
- 是一次性收费、订阅、广告、抽成，还是企业采购？
    
- 毛利率如何？
    
- 获客成本高不高？
    
- 用户生命周期价值高不高？
    
- 规模越大，成本是下降还是上升？
    
- 它有没有网络效应或数据飞轮？
    

尤其要注意：  
有些科技产品“用户很多，但赚不到钱”；有些产品“技术普通，但商业闭环很好”。

# 七、它的护城河是什么？

这是判断长期价值的关键。

可以问：

- 它的优势别人容易复制吗？
    
- 它靠技术领先、数据积累、品牌、渠道、生态，还是成本优势？
    
- 用户迁移成本高不高？
    
- 有没有专利、标准、供应链、监管牌照？
    
- 它越多人用，会不会越强？
    
- 大公司做同类产品时，它还能不能活？
    

常见护城河包括：

**技术壁垒、数据壁垒、生态壁垒、规模效应、品牌信任、渠道优势、监管资质、用户习惯。**

# 八、它的限制和风险是什么？

科技产品往往被高估，所以一定要看反面。

可以问：

- 它在哪些场景下不好用？
    
- 成本是否太高？
    
- 可靠性是否不足？
    
- 是否存在安全、隐私、伦理问题？
    
- 是否依赖单一供应商？
    
- 是否容易被政策限制？
    
- 用户是否真的愿意改变习惯？
    
- 它有没有被宣传过度？
    
例如 VR/AR 很多年都被看好，但长期受限于设备重量、内容生态、价格、眩晕感和使用场景。
# 九、它处在什么发展阶段？

技术产品的价值和阶段强相关。

可以问：

- 它现在是概念验证、早期产品、快速增长、成熟竞争，还是衰退替代？
    
- 技术是否已经稳定？
    
- 成本是否下降到可规模化？
    
- 用户教育是否完成？
    
- 市场是否已经出现明确赢家？
    
- 生态是否形成？
    
- 监管是否清晰？
    
很多技术不是“不行”，而是“太早”。
# 十、它会如何影响行业？

技术产品真正重要时，通常会改变结构。

可以问：

- 它会提高谁的效率？
    
- 它会替代谁？
    
- 它会创造哪些新岗位或新需求？
    
- 它会压缩哪一环的利润？
    
- 它会让行业集中，还是让小团队更容易进入？
    
- 它会改变用户行为吗？
    
- 它会成为基础设施吗？
    

例如云计算不只是“服务器搬到云上”，它改变了软件开发、创业成本、企业 IT 架构和商业模式。
