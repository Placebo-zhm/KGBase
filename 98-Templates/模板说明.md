# 模板说明

## 使用原则

所有正式笔记建议使用统一 Frontmatter，方便后续用 Obsidian Dataview、搜索、属性面板和自动化脚本管理。

基础格式：

```yaml
---
type: note
status: draft
created: {{date}}
updated: {{date}}
tags: []
---
```

## type 字段

`type` 表示笔记类型，推荐值：

```text
inbox
project
area
resource
permanent
moc
review
```

使用建议：

- `inbox`：临时收集，还没有整理完成。
- `project`：有明确目标和完成状态的项目笔记。
- `area`：长期维护的领域笔记。
- `resource`：资料、课程、论文、工具文档等参考内容。
- `permanent`：经过理解和重写的永久笔记。
- `moc`：主题索引页或内容地图。
- `review`：复盘、总结、阶段回顾。

## status 字段

`status` 表示当前状态，推荐值：

```text
draft
active
done
archived
```

使用建议：

- `draft`：草稿，还在整理。
- `active`：正在使用、推进或维护。
- `done`：已经完成。
- `archived`：已经归档，不再主动维护。
