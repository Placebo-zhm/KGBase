# 第 1 个月：深度学习基础 + PyTorch 基础

## 本月目标

把深度学习的核心训练流程彻底搞懂，不再停留在“会调库”。

    - 前向传播
        
    - 反向传播

    - loss / optimizer / scheduler
        
- PyTorch 基础：
    
    - Tensor
        
    - autograd
        
    - Dataset / DataLoader
        
    - Module
        
    - 训练循环与验证循环
        

PyTorch 官方文档把它定位为面向 CPU/GPU 的深度学习张量库与生态入口，适合从基础训练一直延伸到后面的分布式和性能分析。([PyTorch 文档](https://docs.pytorch.org/?utm_source=chatgpt.com "PyTorch documentation — PyTorch 2.11 documentation"))

# 第 5 个月：分布式训练 + 训练系统基础

## 本月目标

理解多卡训练的基本逻辑，知道 DDP/FSDP 在干什么，不再把分布式训练当黑盒。

PyTorch 的 `torch.distributed` 提供多进程并行与通信原语；DDP 通过同步各个模型副本的梯度实现数据并行；FSDP 则通过分片参数、梯度和优化器状态来降低显存占用。([PyTorch 文档](https://docs.pytorch.org/docs/stable/distributed.html?utm_source=chatgpt.com "Distributed communication package - torch.distributed"))

## 你要掌握的内容

- 单卡训练 vs 多卡训练
    
- data parallel 基本思想
    
- `torch.distributed`
    
- DDP
    
- 混合精度训练
    
- 梯度累积
    
- FSDP 的基本概念
    
- checkpoint 保存与恢复
    

## 每周任务

### 第 1 周

- 学 DDP 基本概念
    
- 理解 rank、world size、sampler
    
- 跑官方 DDP 教程
    

PyTorch 官方 DDP 教程明确说明，DDP 需要多进程，并通常为每个进程创建一个模型副本。([PyTorch 文档](https://docs.pytorch.org/tutorials/intermediate/ddp_tutorial.html?utm_source=chatgpt.com "Getting Started with Distributed Data Parallel"))

交付物：

- 跑通 2 卡 DDP demo
    
- 写笔记：《DDP 到底在同步什么》
    

### 第 2 周

- 学 `DistributedSampler`
    
- 比较单卡与多卡训练速度
    
- 学混合精度
    

交付物：

- 1 份实验表格：单卡 vs 多卡
    
- 记录 throughput 与显存
    

### 第 3 周

- 学 FSDP
    
- 理解为什么它更省显存
    
- 跑一个小规模实验
    

交付物：

- 1 个 FSDP demo
    
- 1 篇笔记：《DDP 和 FSDP 的区别》
    

### 第 4 周

- 学 checkpoint 与恢复训练
    
- 学训练过程中的故障恢复思路
    
- 复盘训练系统流程
    

交付物：

- 1 个支持 resume 的训练脚本
    
- 月度总结
    

## 本月结束标准

你应该能做到：

- 讲清 DDP 的基本工作方式
    
- 知道多卡训练为什么不一定线性加速
    
- 知道 FSDP 为什么适合更大模型

# 第 6 个月：GPU 架构、CUDA、Profiler、性能优化入门

## 本月目标

进入“底层理解”阶段，开始形成 GPU 与系统优化视角。

NVIDIA 的 CUDA Programming Guide 是 CUDA 编程模型、架构抽象和性能建议的官方核心资料；NCCL 官方文档则说明了多 GPU 通信原语与拓扑感知通信；PyTorch Profiler 用于测量算子时间和显存消耗。([NVIDIA Docs](https://docs.nvidia.com/cuda/pdf/CUDA_C_Programming_Guide.pdf?utm_source=chatgpt.com "CUDA C++ Programming Guide"))

## 你要掌握的内容

- GPU 与 CPU 的差异
    
- CUDA 编程模型：
    
    - thread
        
    - block
        
    - grid
        
    - warp
        
- 显存层次：
    
    - global memory
        
    - shared memory
        
    - register
        
- memory coalescing
    
- kernel 基本概念
    
- profiler 的使用
    
- NCCL 和 all-reduce 的基本理解
    

NCCL 官方文档指出，它提供拓扑感知的 GPU 间通信原语，包括 collective 与点对点通信，常用于多 GPU 场景。([NVIDIA Docs](https://docs.nvidia.com/deeplearning/nccl/user-guide/docs/overview.html?utm_source=chatgpt.com "Overview of NCCL"))

## 每周任务

### 第 1 周

- 学 GPU 架构基础
    
- 看 CUDA 编程模型
    
- 理解 kernel launch 过程
    

交付物：

- 笔记：《GPU/CUDA 基本概念整理》
    

### 第 2 周

- 写 1～2 个简单 CUDA 例子
    
- 比如向量加法、矩阵乘法
    

交付物：

- CUDA 小实验代码
    
- 记录 CPU vs GPU 的差异
    

### 第 3 周

- 学 PyTorch Profiler
    
- 分析自己的一个训练脚本
    
- 看时间花在哪、显存花在哪
    

PyTorch Profiler 官方教程专门用于测量模型算子的时间与内存消耗。([PyTorch 文档](https://docs.pytorch.org/tutorials/recipes/recipes/profiler_recipe.html?utm_source=chatgpt.com "PyTorch Profiler"))

交付物：

- 1 份 profiling 报告
    
- 至少包含：
    
    - 数据加载耗时
        
    - 前向耗时
        
    - 反向耗时
        
    - 显存占用
        

### 第 4 周

- 学 NCCL、all-reduce 的基本概念
    
- 理解通信为什么会变成瓶颈
    
- 把前 6 个月做一次全面复盘
    

交付物：

- 1 篇总总结：《我对大模型训练系统的初步理解》
    
- 整理 GitHub 仓库主页
    

## 本月结束标准

你应该能做到：

- 看懂 GPU/CUDA 的核心术语
    
- 会用 profiler 找训练慢的原因
