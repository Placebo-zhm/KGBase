## 1. 接收填谷指令

调价平台接收上级平台下发的填谷指令。

填谷指令表示上级平台希望在指定时段内，通过价格引导提升充电电量，完成填谷响应。

| 字段    | 说明            |
| ----- | ------------- |
| 时段    | 本次填谷任务的执行时间   |
| 目标响应量 | 希望通过调价带来的新增电量 |
| 激励总量  | 上级平台计算的激励金额   |

## 2. 根据敏感度筛选可参与充电站

平台基于站点价格敏感度和可参与条件，从平台管理的充电站中筛选适合参与填谷调价的站点。

筛选依据包括：

| 筛选条件  | 说明                    |
| ----- | --------------------- |
| 价格敏感度 | 用户对站点价格变化的响应程度        |
| 基线电量  | 站点在不调价情况下的预测充电电量      |
筛选结果形成可参与站点池：

```
可参与充电站集合 = {站点1，站点2，站点3，...，站点n}
```

## 3. 根据敏感度生成候选调价动作

平台根据每个站点的价格敏感度，为站点生成候选调价动作。

价格敏感度越高，说明用户对该站点价格变化越敏感，降价后更容易带来新增电量。

因此，不同站点生成的候选价格可以不同：

| 站点类型  | 调价动作生成逻辑             |
| ----- | -------------------- |
| 高敏感站点 | 可生成多个降价档位，适合作为重点填谷站点 |
| 中敏感站点 | 生成适中的降价档位，用于补充响应量    |
| 低敏感站点 | 少量生成或不生成调价动作，避免无效让利  |
例如：

| 站点  | 当前价格 | 候选价格 |
| --- | ---: | ---: |
| A站  | 0.60 | 0.55 |
| A站  | 0.60 | 0.50 |
| A站  | 0.60 | 0.45 |
| B站  | 0.58 | 0.52 |
| B站  | 0.58 | 0.48 |

## 5. 形成候选动作表

平台形成候选动作表。
候选动作表是后续组合优化的输入。表中每一行代表一个站点在一个候选价格下的预测结果。
收益由两部分构成：

```
收益 = 服务费收益 + 电网补贴收益
```

其中**服务费收益**为

```text
服务费收益 = 调价后电量 × 谷段服务费单价
```

**补贴收益**定义为：

```
电网补贴收益

有效补贴响应量 = 
	if 实际响应量 < 下限阈值 × 上报响应量: 
		0 
	else: 
		min(实际响应量, 上限阈值 × 上报响应量)
		
其中

超额响应量 = max(实际响应量 - 上限阈值 × 上报响应量, 0)

超额响应量不计入电网补贴收益，但仍可能产生用户充电服务费收入

```

若**响应成功**，即实际响应量达到上报响应量，则平台可获得对应的电网补贴或电费减免收益。

补贴收益需考虑**响应上下限**，超过下限才有补贴，超过上限部分不享受补贴

若**响应失败**，即实际响应量未达到上报响应量的下限，则平台不仅无法获得响应收益，还需承担电价差额损失。因此，模型中需要增加风险损失计算因子。

风险损失定义为：

```
风险损失 = 未完成响应电量 ×（尖峰电价 a - 谷段电价 b）
```

其中：

| 字段      | 说明                 |
| ------- | ------------------ |
| 尖峰电价 a  | 响应失败时系统需承担或结算的高价电价 |
| 谷段电价 b  | 用户实际按谷段费率充电对应的低价电价 |
| 未完成响应电量 | 上报响应量与实际完成响应量之间的差值 |

```
未完成响应电量 = max(上报响应量 - 实际响应量, 0)
```

因此，候选动作的综合收益可表示为：

```
综合收益 = 服务费收益 + 电网补贴收益 - 风险损失
``````

示例：

| 站点  | 推荐价格 | 基线电量 | 调价后电量 | 响应量 | 服务费收益 | 电网补贴收益 | 风险损失 | 综合收益 |
| --- | ---- | ---- | ----- | --- | ----- | ------ | ---- | ---- |
| A站  | 0.55 | 800  | 920   | 120 | 55    | 40     | 0    | 95   |
| A站  | 0.50 | 800  | 1050  | 250 | 95    | 80     | 20   | 155  |
| A站  | 0.45 | 800  | 1280  | 480 | 120   | 130    | 60   | 190  |
| B站  | 0.52 | 500  | 680   | 180 | 70    | 60     | 0    | 130  |
| B站  | 0.48 | 500  | 810   | 310 | 90    | 90     | 30   | 150  |
| C站  | 0.55 | 300  | 420   | 120 | 40    | 35     | 0    | 75   |

候选动作表包含以下核心字段：

```
站点 / 推荐价格 / 基线电量 / 调价后电量 / 响应量 / 服务费收益 / 电网补贴收益 / 风险损失 / 综合收益
```

## 6 优化目标

在所有可能组合中选择：

```text
收益最高
+
响应量最接近目标
+ 
响应失败风险最低
```

的站点调价组合。

## 7. 输出填谷调价推荐方案

组合优化完成后，平台输出最终的 **填谷调价推荐方案**。

方案包括：

| 字段    | 说明              |
| ----- | --------------- |
| 推荐站点  | 本次建议参与填谷调价的充电站  |
| 推荐价格  | 每个站点在填谷时段内的推荐价格 |
| 基线电量  | 每个站点不调价时的预测电量   |
| 调价后电量 | 每个站点调价后的预测电量    |
| 响应量   | 调价后电量与基线电量的差值   |
| 收益    | 每个站点调价动作对应的预计收益 |
