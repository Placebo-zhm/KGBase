### 第一阶段：先把深度学习和 PyTorch 打扎实

这一阶段的目标，不是“看很多论文”，而是做到：

- 能独立写 MLP/CNN/RNN/Transformer
- 理解 forward、backward、autograd、loss、optimizer、scheduler
- 知道过拟合、正则化、归一化、初始化、梯度爆炸/消失
- 能自己写训练循环、验证循环、日志与可视化

你必须先把 **PyTorch 训练范式**练熟，因为后面大模型训练、分布式训练、性能优化，全都是在这个基础上升级。PyTorch 官方文档把它定位为面向 GPU/CPU 的深度学习张量库，而 profiler 还能帮助你分析训练和推理时最耗时、最耗显存的算子。

这一阶段你最该做的，不是追热点，而是自己写 3 个完整小项目：

- 图像分类
- 文本分类
- 一个小型 Transformer 训练实验

只要你能把这 3 个项目从数据读取到训练、评估、保存模型都独立跑通，底子就稳了。

## 三、第二阶段：进入大模型开发，但先做“会微调和会评估的人”

很多人一上来就想自己训 LLM，这对研究生前期不划算。更合理的是先学：

- Hugging Face Transformers
- tokenizer、dataset、collator
- SFT/指令微调
- PEFT / LoRA
- evaluation 与 error analysis
- inference 与 prompt format

Transformers 是目前最主流的模型开发框架之一，覆盖文本、视觉、音频、视频、多模态模型；PEFT 则专门解决“大模型全量微调太贵”的问题，通过只训练少量附加参数来降低算力和存储成本。

你这一阶段的正确目标是：

**先成为一个能把开源模型“改好、训好、评估好、部署好”的人。**

建议你做 2 类项目：

1. **领域微调项目**：比如法律、医疗、科研问答、小型中文任务
2. **研究增强项目**：比如把你自己的研究方向接入大模型流程中，做一个“AI for Science”或“领域智能助手”

你现在是研究生，这一步最加分，因为它既能写进简历，也容易自然衔接论文选题。

## 四、第三阶段：开始补“训练系统”能力，这是你和普通算法同学拉开差距的关键

当你会单卡训练后，就要开始理解多卡和显存优化。

PyTorch 的 **DDP** 本质上是每个进程/设备维护一份模型副本，然后通过梯度同步来完成数据并行；而 **FSDP** 会对参数、梯度和优化器状态做分片，从而显著降低显存占用，更适合更大的模型。PyTorch 还提供了分布式 checkpoint，用于多 rank 并行保存和跨拓扑加载。

所以你的顺序应该是：

**单卡训练 → DDP → 混合精度 → 梯度累积 → FSDP → checkpoint/恢复训练**

这一阶段你需要掌握的不是“API 会用”，而是这些问题：

- 为什么多卡不一定更快？
- 通信什么时候成为瓶颈？
- batch size、显存、吞吐之间怎么平衡？
- 为什么训练会 OOM？
- 为什么 save/load 在多机多卡下更复杂？

这一步一旦学会，你就不再只是“模型使用者”，而是开始具备**训练系统工程师**的底子。
