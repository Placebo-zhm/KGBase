

## 概述

### 1. 项目概况

随着新能源汽车保有量持续提升，公共充电站的用电负荷呈现出强时段性、强波动性和站点差异化特征。充电需求受历史使用习惯、节假日、周末、天气、电价机制、周边交通及站点服务能力等多因素影响，传统依靠人工经验或固定阈值的运行监测方式难以及时识别未来负荷高峰和过载风险。

本项目围绕“多尺度用电需求预测与过载风险预警”目标，建设面向充电站运行管理的预测预警平台。系统以充电站订单数据为核心数据源，通过订单交易电量、充电开始时间、结束时间反推站点 15 分钟级功率序列；在此基础上融合日周期、周周期、节假日、周末等时间特征，构建基于 PatchTST 的站点负荷预测模型，输出未来 15 分钟粒度的负荷预测曲线，并结合历史基线形成风险等级评估、风险成因解释和分级应急处置方案展示。

项目输出为充电站电量预测预警平台，支持多时间尺度负荷预测、综合预测误差评估、过载风险监测和不少于 10 种差异化应急响应方案，为充电站运行调度、负荷削峰、用户分流和应急处置提供数据支撑。

## 二、设计思路、原则、方案及关键技术

### 1. 控制目标范围

1. 时间目标：围绕近一年充电站订单数据，构建 15 分钟级功率时间序列，支持按天生成未来负荷预测结果，并可扩展至小时级、日级预测展示。
2. 质量目标：以 MAE、MSE、MAPE 等指标评估模型效果，支撑负荷预测误差不高于 10% 的目标要求；
3. 集成目标：后端提供标准 REST API，支持前端、地图、图表、方案库等模块调用；预测结果、风险等级、特征贡献和响应方案统一写入数据库并可重复查询。
4. 运行目标：模型优先复用已训练权重和元数据，缺失模型时自动训练并保存，降低重复计算成本；预测任务按站点逐一执行，单站异常不影响其他站点结果返回。

### 2. 多源数据融合与负荷序列构建

项目当前工程实现以充电站订单数据和日期时间特征为核心输入，后续可继续接入天气、电价和单桩实时功率数据。数据用途如下：

| 数据类别    | 关键字段                | 粒度         | 用途                      |
| ------- | ------------------- | ---------- | ----------------------- |
| 充电站订单数据 | 交易电量、开始时间、结束时间、站点编号 | 单笔订单级      | 反推单次订单平均功率，构建站点历史负荷曲线   |
| 节假日数据   | 节假日标识、工作日/周末标识      | 日级         | 描述日期属性差异，提升节假日和周末负荷识别能力 |
| 天气数据    | 气温、降雨量等             | 15 分钟级或小时级 | 作为扩展环境特征，刻画天气对充电需求的影响   |
| 电价数据    | 分时电价、计费模型           | 分时段        | 分析价格机制对用户充电行为和负荷分布的影响   |
| 充电桩功率数据 | 单桩实时功率、站级汇总功率       | 15 分钟级     | 支撑实时监测、风险校核和动态调控        |

后端根据订单构建 15 分钟粒度功率序列。形成站点级负荷曲线。该方法能够在缺少单桩实时功率采样数据时，利用订单数据近似重构站点历史负荷水平，为模型训练、基线计算和预测结果校验提供统一数据基础。

### 3. 特征工程

系统统一生成时间特征。当前工程已实现以下特征：

| 特征 | 含义 |
|---|---|
| `kw` | 历史负荷功率，是模型预测目标的历史窗口输入 |
| `tod_sin`、`tod_cos` | 日内分钟的正余弦编码，刻画一天内充电需求周期 |
| `dow_sin`、`dow_cos` | 周内日期的正余弦编码，刻画工作日与周末规律 |
| `is_weekend` | 周末标识 |
| `is_holiday` | 中国节假日标识，依赖 `chinese-calendar` |

时间特征采用正余弦周期编码，避免小时、星期等离散数值在周期边界处产生不连续问题。例如 23:45 与 00:00 在业务上相邻，但直接使用分钟值会导致数值距离过大；正余弦编码可以保持周期变量的连续性。

数据集构建流程包括：按时间排序、生成时间特征、按时间比例切分训练集/验证集/测试集、仅使用训练集统计量进行标准化、按滑动窗口生成模型样本。当前模型输入形状为：

```text
X = (样本数, lookback, 特征数)
Y = (样本数, horizon)
```

其中 `lookback=288`，表示 15 分钟粒度下回看 3 天历史窗口；`horizon=96`，表示输出未来 1 天共 96 个预测点。工程中保留了多时间尺度扩展能力，可按需要调整为小时级或日级聚合展示。

### 4. 多尺度负荷预测模型

项目采用 PatchTST 作为核心预测模型。PatchTST 将长时间序列切分为多个时间片段 patch，通过 Transformer Encoder 建模局部片段与长距离依赖关系，适合处理充电站负荷这种具有日周期、周周期和局部尖峰波动的时间序列。

模型主要结构如下：

1. 输入层：接收历史负荷与时间特征组成的多变量时间序列。
2. Patch 切分：按 `patch_len=30`、`stride=30` 将历史窗口切分为若干局部片段。
3. 线性嵌入：将每个 patch 映射到 `d_model=128` 的隐空间。
4. 变量嵌入与位置编码：为变量维度和时间片段注入可学习变量信息与正弦位置编码。
5. Transformer Encoder：通过多头注意力捕捉不同时间片段之间的相关关系。
6. 输出头：将编码后的全部 token 展平，通过全连接网络输出未来 `horizon` 个功率预测值。

核心参数如下：

| 参数 | 当前配置 | 说明 |
|---|---:|---|
| 预测粒度 | 15min | 每天 96 个点 |
| 历史窗口 | 288 点 | 回看 3 天 |
| 预测步长 | 96 点 | 预测未来 1 天 |
| 训练历史 | 最多 365 天 | 从预测开始时刻向前取历史订单 |
| 训练轮数 | 100 | 默认训练轮数 |
| 批大小 | 16 | 模型训练批大小 |
| 学习率 | 1e-4 | Adam 类优化器学习率 |

### 5. 基线计算与风险等级评估

系统在预测输出后同步计算历史基线。基线采用预测日前最近一周同一日内时段的平均功率，按 15 分钟槽位聚合：

```text
baseline(slot) = 最近一周历史数据中同一 slot 的平均负荷
```

风险等级通过预测平均功率与基线平均功率的相对偏差进行评估：

```text
risk_ratio = (avg_forecast_power - avg_baseline_power) / avg_baseline_power
```

当前后端实现的风险等级为：

| 风险等级 | 工程判定口径          |
| ---- | --------------- |
| 一级   | 偏离率 ≤ 10%       |
| 二级   | 10% < 偏离率 ≤ 30% |
| 三级   | 30% < 偏离率 ≤ 50% |
| 四级   | 偏离率 > 50%       |

### 6. 特征贡献与风险成因分析

为提升预测结果可解释性，系统引入 SHAP 特征贡献分析。后端对平均预测功率进行解释，输出每类输入特征对预测结果的贡献值和平均绝对贡献值，前端风险信息面板读取 SHAP 结果，以环形图展示风险成因占比。该设计使运行人员不仅能看到站点风险等级，还能进一步判断风险主要来自历史负荷惯性、日内高峰、周末效应或节假日影响。

### 7. 分级响应机制与应急方案库

项目建立四级响应机制，并设计不少于 10 种差异化应急处置方案。

| 响应阶段      | 适用场景                 | 对应方案                                        |
| --------- | -------------------- | ------------------------------------------- |
| 一级：关注阶段   | 负荷偏离基线，处于早期异常阶段      | 方案1 双向监测与异常留痕；方案2 趋势复核与滚动预测                 |
| 二级：预警阶段   | 偏离持续扩大，需要运营提醒和处置准备   | 方案3 预警信息推送；方案4 预警状态切换                       |
| 三级：高风险阶段  | 负荷持续处于高位，需要调控充电行为    | 方案5 充电桩功率限制；方案6 分组轮控；方案7 动态功率重分配；方案8 轻载运营调节 |
| 四级：严重风险阶段 | 站点运行风险较高，需要强制压降或异常核查 | 方案9 全站整体压降；方案10 暂停新增会话与轻载异常核查               |
## 方案细节
### 一级：关注阶段

#### 方案1 双向监测与异常留痕

加强对重载、轻载异常的监测，完整记录负荷偏离率、持续时长、枪位占用率、空闲率、等待时长、启动成功率、设备在线率等信息，为后续判断和处置提供依据。

#### 方案2 趋势复核与滚动预测

结合未来1～2小时负荷预测、历史同类型日、天气、节假日、周边交通和站点开放状态等因素，判断当前负荷异常是短时波动，还是已形成持续上升或持续下降趋势。

### 二级：预警阶段

#### 方案3 预警信息推送

系统将重载或轻载预警信息推送至运营人员、站点管理人员或调度人员。运营人员接到预警后，可视情况向用户发布繁忙提醒、空闲站点推荐、建议到站时间、替代站点或错峰引导信息。

#### 方案4 预警状态切换

站控系统切换至预警模式，同步启动异常日志、预测复核、风险滚动评估和运营处置准备机制，进入预警处置状态。

### 三级：高风险阶段

#### 方案5 充电桩功率限制

重载高风险时，对部分非核心充电枪下调输出功率上限，在不停运的前提下优先压降局部高负荷，防止负荷继续抬升。

#### 方案6 分组轮控

重载高风险时，将充电枪划分为不同运行组，采用交替高低档输出方式，降低同一时段集中功率峰值。

#### 方案7 动态功率重分配

重载高风险时，优先保障低SOC、急需补能车辆的充电需求，对后段充电车辆适当降档，提高有限容量利用效率。

#### 方案8 轻载运营调节

轻载高风险时，启动空闲站点推荐、错峰优惠、周边重载站点分流、设备巡检、平台展示核查和低利用原因排查等措施，提高站点利用率并识别潜在运营异常。

### 四级：严重风险阶段

#### 方案9 全站整体压降

重载严重风险时，对全站在运充电枪统一降额运行，并设置全站总功率上限，快速将总负荷拉回可控区间。

#### 方案10 暂停新增会话与轻载异常核查

重载严重风险时，暂停新的充电会话接入，已在充电会话维持受控运行，防止负荷继续抬升。轻载严重风险时，核查设备、通信、支付、导航、站点开放、停车限制等异常，并根据核查结果调整站点推荐或派发故障工单。

### 1. 总体架构

系统采用前后端分离架构，整体链路如下：

```text
充电站订单数据 / 站点数据
        ↓
后端 FastAPI 服务
        ↓
订单查询、功率序列构建、特征工程
        ↓
PatchTST 训练/加载/预测
        ↓
基线计算、风险评估、SHAP 成因分析
        ↓
预测结果与风险方案入库
        ↓
REST API
        ↓
Vue 前端可视化大屏
```

后端负责数据处理、模型推理、数据库读写和接口输出；前端负责模型参数操作、站点选择、地图展示、图表展示、风险解释和应急方案展示。

后端项目主要目录如下：

```text
app/

  core/

  power_forecast/

```

核心数据表包括：

| 表/模型 | 说明 |
|---|---|
| `charging_station` | 充电站基础信息 |
| `order` | 充电订单信息，用于反推历史功率 |
| `power_forecast_result` | 保存预测曲线、基线、实际值、风险等级、指标和 SHAP 结果 |

后端主要接口包括：

| 接口 | 方法 | 功能 |

## 四、功能介绍

### 1. 站点负荷数据管理

系统面向充电站维度组织数据。站点基础信息用于地图展示和预测结果归属，订单数据用于构建历史负荷序列。后端通过固定时间窗口查询与预测区间相关的订单，既覆盖模型训练所需历史数据，也覆盖实际功率对比所需的预测区间数据。

前端当前以 15 分钟为默认展示窗口，定时调用平均功率接口刷新站点总负荷，用于模拟实时负荷监测效果。该机制后续可替换为真实实时功率采集接口或 WebSocket 推送。

### 2. 模型参数设置与预测执行

前端 `ModelControlPanel` 提供模型选择、数据集时间范围、预测时间段、预测步长和特征选择控件。当前默认调用 PatchTST 模型，并以 `2026-01-30 00:00:00` 至 `2026-01-30 23:45:00` 为预测演示区间。

预测执行时，前端调用后端 `POST /api/v1/power-forecast`。后端对数据库中的所有充电站逐一预测，并返回每个站点的执行状态、保存点数、风险等级、风险比例、SHAP 状态和模型误差指标。若某一站点模型缺失、数据不足或预测失败，系统会记录失败原因并继续处理其他站点。

### 3. 负荷预测曲线展示

前端 `LoadForecastChart` 使用 ECharts 折线图展示三类曲线：

1. 实际负荷：来自预测区间内已有订单反推的实际功率，未来时间可能为空。
2. 预测负荷：PatchTST 输出的未来功率预测值。
3. 基线负荷：最近一周同一时段的平均负荷。

三类曲线能够帮助用户同时观察历史真实水平、未来预测走势和风险判定参考基线。当预测曲线明显高于基线曲线时，系统会在风险面板和地图点位中同步体现风险等级变化。

### 4. 过载风险预警展示

前端 `RiskInfoPanel` 汇总站点风险信息，展示站点名称、风险等级、当前负荷、预测峰值和预测峰值时段。风险等级以颜色和图标区分，便于运行人员在大屏中快速识别重点站点。

风险峰值时段通过滑动窗口计算预测功率最高的一段连续时间，当前前端默认以 4 个 15 分钟点作为高负荷窗口，即约 1 小时高峰时段。该信息可用于安排提前分流、限功率或现场巡检。

### 5. 风险成因分析

系统将 SHAP 输出特征按业务类别归并并展示占比。例如，当“历史负荷特征”占比较高时，说明站点当前风险主要来自历史负荷惯性和近期持续高需求；当“日周期特征”占比较高时，说明风险与固定日内高峰时段相关；当“节假日”或“周末”贡献较高时，说明特殊日期属性对负荷抬升有明显影响。

该功能提升了模型预测的可解释性，有助于运行人员区分“周期性高峰”“节假日集中需求”“持续异常增长”等不同风险类型。

### 6. 应急方案联动

前端 `EmergencyPlanPanel` 读取后端返回的 `response_plans`，按当前风险等级展示对应处置方案。平台支持从一级关注到四级严重风险的分级处置，包括监测留痕、趋势复核、预警推送、状态切换、局部限功率、分组轮控、动态功率重分配、轻载运营调节、全站压降和暂停新增会话等。

该设计将“预测、识别、解释、处置”串联为闭环流程，使平台不仅展示风险，还能给出可执行的处置建议。

## 五、系统特点

1. 数据驱动：基于充电订单构建站点级 15 分钟负荷序列，在缺少实时功率数据时仍可支撑预测建模。
2. 多尺度扩展：当前支持 15 分钟级预测，系统结构可扩展至小时级、日级聚合展示。
3. 模型先进性：采用 PatchTST 对长时间序列进行 patch 化建模，兼顾局部波动和长期周期规律。
4. 风险闭环：预测结果自动生成基线偏离率和风险等级，并联动应急方案库。
5. 可解释性：通过 SHAP 分析展示风险成因占比，提升模型结果可信度。
6. 前后端解耦：后端提供标准 REST API，前端通过统一接口封装调用，便于后续接入更多终端。
7. 可视化友好：集成地图、折线图、环形图、指标卡和方案列表，适配运行监控大屏使用场景。

## 六、存在的需要改进的地方

1. 天气与电价特征接入：PPT 方案中规划了天气、电价等多源特征，当前工程主要使用历史负荷和日期特征，后续应补充气温、降雨量、分时电价等外部因子。
2. 实时数据闭环：当前平均功率刷新以订单反推为主，后续建议接入充电桩实时功率、枪位占用率、启动成功率和设备在线率，提高实时监测精度。
3. 模型滚动优化：当前系统支持模型缺失时训练和已有模型加载，后续可增加按日滚动训练、漂移检测和在线校准机制。
4. 应急执行接口：当前平台展示处置方案，尚未直接打通小程序推送、短信推送、站控系统限功率、暂停新增会话等执行接口，后续可按业务权限逐步接入。
5. 轻载风险量化：方案库已包含轻载运营调节与异常核查内容，后续可进一步建立轻载风险指标，如空闲率、利用率、导航曝光、设备故障率等。

## 七、结论

本项目围绕充电站负荷预测和过载风险预警，完成了从订单数据建模、15 分钟级功率序列构建、PatchTST 预测、历史基线对比、风险等级评估、SHAP 成因分析到应急方案联动展示的完整技术链路。系统实现了后端智能预测服务与前端可视化大屏的协同，为充电站运营管理提供了可查询、可解释、可处置的预测预警能力。

后续通过补充天气、电价和实时单桩功率数据，完善风险阈值配置、执行接口和滚动优化机制，可进一步提升平台在真实充电站运行场景中的预测准确性、风险响应速度和调度处置能力。
