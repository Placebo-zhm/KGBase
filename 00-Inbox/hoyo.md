那你应该按**游戏客户端 / 引擎图形 / 游戏 AI / 后端服务**这几条线来准备。米哈游技术岗不是单纯刷题型公司，它更看重：**扎实 CS 基础 + 游戏工程经验 + 能做出高完成度 Demo + 对内容生产有理解**。HoYoverse 近期公开技术岗位也明确提到 C++/C#、数据结构、算法、操作系统、网络、UE4/UE5、游戏 AI、性能优化、服务端高并发、AIGC/LLM/Agent 等方向。

## 你最该优先选的方向

如果你还没有特别强的偏科，我建议优先走：

**客户端 / 玩法工程 > 游戏 AI > 引擎图形 > 后端服务**

原因很简单：客户端/玩法工程离“游戏产品”最近，作品集最好做，也最容易让面试官看到你的潜力。米哈游公开技术介绍里也长期强调卡通渲染、云游戏、AI 等技术能力；这些方向很香，但门槛更高，尤其图形和引擎需要更硬的数学、底层和工程积累。

## 路线 1：客户端 / 玩法工程，最推荐

你要掌握：

- **C++ 或 C#**：至少一门非常熟，别只是“写过”
- **Unity 或 Unreal**：现在建议优先 Unreal / C++，但 Unity / C# 也能投不少客户端岗
- **游戏基础模块**：角色控制、相机、技能系统、Buff、动画状态机、UI、资源加载、存档、战斗结算
- **性能优化**：帧率、GC、DrawCall、内存、加载耗时、对象池
- **工程能力**：模块拆分、配置表、事件系统、状态机、可扩展技能框架

你最好做一个这样的项目：

> **3D 动作战斗 Demo**  
> 包含角色移动、锁定、闪避、普攻连段、技能释放、受击、怪物 AI、简单关卡、UI、配置表、性能说明。

这个项目比“我做了一个小游戏”有用太多。米哈游的技术岗位会关注你能不能把设计、美术、工程协作落到稳定代码里；近期 Game AI 岗位描述里也写到，需要和设计、美术、工程合作，把玩法设计转化成稳定、高性能代码。

## 路线 2：游戏 AI，很适合做差异化

如果你对 NPC、战斗怪物、行为决策感兴趣，可以走这个方向。近期 HoYoverse 的 Game AI Engineer 岗位明确提到 NPC 行为、决策、寻路、感知系统、行为树、状态机、A*、NavMesh、UE 中 AI 模块开发与优化。

你可以做一个：

> **小型开放场景 NPC AI Demo**  
> 包含巡逻、视野感知、听觉感知、追击、搜索、撤退、攻击、群体协作、NavMesh 寻路、行为树可视化调试。

加分项是写一篇技术文档：  
**“我的 AI 系统如何设计、如何调试、性能瓶颈在哪、如何扩展新怪物行为。”**

这个比单纯说“我了解行为树”强十倍。

## 路线 3：图形 / 渲染 / TA，难但很米哈游

米哈游的产品非常吃美术表现，公开资料也提到卡通渲染等技术积累。  
如果你想走图形方向，需要准备：

- 线性代数、图形学管线、光照模型、PBR、阴影、后处理
- Shader 编写，HLSL / GLSL
- Toon Shading / NPR
- GPU 性能分析
- Unity URP/HDRP 或 Unreal Render Pipeline
- 工具开发能力

作品建议：

> **卡通渲染角色展示 Demo**  
> 包含描边、Ramp 光照、头发高光、脸部阴影控制、边缘光、后处理、性能分析。

这条线作品集很重要。别只放截图，要放：  
**效果图 + Shader 代码 + 参数说明 + 性能数据 + 技术拆解。**

## 路线 4：后端 / 平台 / 高并发

如果你算法和计算机基础更强，可以走服务端。近期岗位描述里也提到游戏服务端开发、高并发玩家负载、稳定性和在线运营支持。

你要准备：

- Go / C++ / Java 至少一门
- 网络编程、RPC、数据库、缓存、消息队列
- 分布式系统基础
- 高并发、限流、熔断、容灾
- 游戏房间、匹配、战斗服、登录、背包、排行榜、活动系统

项目建议：

> **多人在线战斗/房间服务 Demo**  
> 包含登录、匹配、房间、状态同步、断线重连、排行榜、压测报告。

一定要有压测数据。比如：  
“单机支持 5,000 并发连接，平均响应 xx ms，使用 Redis 做排行榜缓存。”  
这种东西很能打。

## 你现在最该补的四块基础

**第一，数据结构和算法。**  
不是为了刷题本身，而是因为游戏里到处都是图、树、状态、搜索、排序、缓存。LeetCode 中等题能稳定做出来就够，但图搜索、堆、哈希、动态规划、并查集、字符串要熟。

**第二，操作系统和网络。**  
线程、锁、内存、进程、IO、多路复用、TCP/UDP、延迟、丢包、同步，这些面试很容易问。尤其你投客户端或服务端，网络同步和性能问题绕不开。

**第三，C++ / C# 深度。**  
C++ 要懂对象生命周期、RAII、智能指针、虚函数、内存布局、STL、移动语义。  
C# 要懂 GC、委托、事件、协程、反射、泛型、异步、Unity 生命周期。

**第四，游戏引擎实践。**  
不要停在“会用 Unity/UE”。你要能解释：  
“为什么这样拆模块？”  
“性能瓶颈怎么定位？”  
“技能系统怎么支持策划配置？”  
“状态机和行为树怎么选？”  
“资源加载为什么会卡顿？”

## 12 周准备计划

**第 1-2 周：补 C++/C# + 算法基础**  
每天 2 道算法题，重点刷数组、链表、树、图、堆、哈希。  
同时复习语言底层和操作系统。

**第 3-4 周：做战斗 Demo 骨架**  
先做移动、相机、动画、攻击、受击。  
不要一上来追求画面，先让系统跑起来。

**第 5-6 周：做技能系统和敌人 AI**  
技能配置表、冷却、打断、硬直、Buff、怪物状态机。  
这里是项目的含金量核心。

**第 7-8 周：做专项技术亮点**  
比如行为树、寻路、Shader、网络同步、编辑器工具。  
这部分决定你是“普通会写代码”还是“有岗位匹配度”。

**第 9 周：性能优化和工程整理**  
加对象池、资源加载优化、Profiler 截图、帧率数据、内存数据。  
没有性能分析的游戏项目，像没放盐的薯条，能吃但不香。

**第 10 周：写文档和录视频**  
GitHub README、架构图、演示视频、技术难点说明。  
视频控制在 2-3 分钟，开头 10 秒必须展示最强效果。

**第 11 周：简历打磨**  
不要写“熟悉 Unity”。写成：  
“实现基于配置表的技能系统，支持连段、冷却、打断、Buff 叠加和动画事件驱动命中判定。”

**第 12 周：模拟面试**  
准备项目深挖、算法、C++/C#、操作系统、网络、引擎机制。  
米哈游校招官网也提示，不同岗位面试流程会灵活开展；应届生和实习投递次数有限，所以投递前材料要尽量一次打磨到位。
