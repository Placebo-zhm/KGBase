# **G-Retriever: Retrieval-Augmented Generation for**
# **Textual Graph Understanding and**
# **Question Answering**
![[Pasted image 20260604220650.png]]
- **Indexing：图索引**  
    首先，将图中的节点属性和边属性分别输入语言模型，得到节点向量和边向量，并存入向量索引中。
- **Retrieval：相关节点和边检索**  
    当问题输入后，模型将问题编码为查询向量，然后根据语义相似度检索出与问题最相关的节点和边。
- **Subgraph Construction：相关子图构建**  
    根据检索到的节点和边，进一步构建一个连通的相关子图，使其尽量包含有用信息，同时控制子图规模，避免引入过多无关内容。
- **Generation：答案生成**  
    最后，将检索到的子图一方面编码为 graph prompt，另一方面文本化为节点和边列表，再与问题一起输入 LLM，生成最终答案。
