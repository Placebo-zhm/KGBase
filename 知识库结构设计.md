# Obsidian 知识库结构设计

## 设计目标

这个知识库采用 `PARA + MOC` 的组织方式，用来支持个人学习、项目推进、资料沉淀和长期复用。

核心原则：

- 先收集，再整理，不要求每条笔记一开始就放到完美位置。
- 用项目推动行动，用领域维护长期责任，用资源沉淀参考材料。
- 用 MOC（Map of Content，内容地图）建立主题索引，避免只依赖目录层级。
- 旧内容不删除优先归档，保留上下文和演化过程。

## 推荐目录结构

```text
00-Inbox/
10-Projects/
20-Areas/
30-Resources/
40-Archives/
90-MOCs/
98-Templates/
99-Attachments/
```

### 00-Inbox

临时收集区，用于放置还没有整理的想法、摘录、灵感、待处理材料。

适合放入：

- 临时想法
- 网页摘录
- 会议或课堂速记
- 还没有判断归属的笔记

使用规则：

- Inbox 不是长期存放区。
- 每周至少清理一次。
- 清理时将内容移动到项目、领域、资源或归档中。

模板参考：[[98-Templates/临时笔记模板|临时笔记模板]]

### 10-Projects

项目区，用于放置有明确目标、阶段、完成状态的内容。

适合放入：

- 正在完成的课程任务
- 论文、实验、开发任务
- 有截止时间的学习计划
- 某个专题输出，如文章、报告、演示文稿

判断标准：

- 有明确结果。
- 有开始和结束。
- 可以被完成、暂停或归档。

模板参考：[[98-Templates/项目笔记模板|项目笔记模板]]

### 20-Areas

领域区，用于放置需要长期维护、没有明确结束时间的方向。

适合放入：

- 专业方向
- 长期技能
- 工作职责
- 健康、财务、写作等长期主题

判断标准：

- 没有明确完成日期。
- 需要持续维护。
- 会不断产生新的项目和资源。

模板参考：[[98-Templates/永久笔记模板|永久笔记模板]]

### 30-Resources

资源区，用于存放主题资料、课程笔记、参考材料和可复用信息。

适合放入：

- 书籍笔记
- 课程资料
- 论文阅读
- 工具用法
- 技术文档摘要

判断标准：

- 当前不一定要执行。
- 以后可能会复用。
- 更偏参考和资料，而不是任务。

模板参考：[[98-Templates/资料笔记模板|资料笔记模板]]、[[98-Templates/论文笔记模板|论文笔记模板]]

### 40-Archives

归档区，用于存放已完成、暂停、过期或暂时不用的内容。

适合放入：

- 完成的项目
- 过期计划
- 不再维护的资料
- 已经整理完的临时材料

使用规则：

- 归档不等于删除。
- 如果内容未来还可能参考，优先归档。
- 如果内容完全无价值，再考虑删除。

模板参考：[[98-Templates/日志模板|日志模板]]

### 90-MOCs

MOC 区用于建立主题索引页。MOC 不一定保存完整内容，它主要负责组织链接、路径和学习脉络。

适合放入：

- 某个领域的导航页
- 某门课程的总索引
- 某个技术主题的知识地图
- 某个项目的入口页

模板参考：[[98-Templates/MOC模板|MOC模板]]

### 98-Templates

模板区用于保存 Obsidian 模板、常用笔记格式和可复用写作框架。

适合放入：

- 临时笔记模板
- 项目笔记模板
- 论文笔记模板
- 资料笔记模板
- 永久笔记模板
- MOC 模板
- 日志模板

使用规则：

- 模板只放结构，不放具体内容。
- 模板文件名应清楚表达用途。
- 常用模板可以在 Obsidian 模板插件中配置为默认模板来源。

模板参考：[[98-Templates/模板说明|模板说明]]

### 99-Attachments

附件区用于统一保存图片、PDF、压缩包和其他非 Markdown 文件。

推荐结构：

```text
99-Attachments/
  images/
  pdfs/
  files/
```

使用规则：

- 图片不要长期散落在根目录。
- Obsidian 设置中建议将附件默认保存位置改为 `99-Attachments/images`。
- 大文件谨慎加入 Git，必要时使用网盘或 Git LFS。

## 命名规范

### 普通笔记

格式：

```text
主题 - 具体内容.md
```

### 项目笔记

格式：

```text
项目名 - 任务或阶段.md
```

### 日志和记录

格式：

```text
YYYY-MM-DD 内容.md
```

### 附件

附件统一放入 `99-Attachments`，不要长期散落在根目录。

推荐命名：

```text
YYYYMMDD-主题-说明.png
YYYYMMDD-主题-说明.pdf
```

## 判断笔记应该放在哪里

可以用以下问题判断：

- 只是临时记录，还没整理？放入 `00-Inbox`。
- 有明确目标和完成状态？放入 `10-Projects`。
- 是长期维护的方向？放入 `20-Areas`。
- 是以后可能复用的资料？放入 `30-Resources`。
- 已完成、暂停或过期？放入 `40-Archives`。
- 是主题入口和导航？放入 `90-MOCs`。
- 是可复用笔记格式或写作框架？放入 `98-Templates`。
- 是图片、PDF 或其他附件？放入 `99-Attachments`。
